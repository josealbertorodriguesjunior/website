What’s in the box?

The launchers let you install different items based on your needs. If you’re trying Subutai for the first time we recommend installing everything to get the full experience. Uninstallation is trivial and nothing stays on your machine.

Peer VM

This installation option creates a small peer on your desktop isolated inside a virtual machine. It’s perfect for non-technical users to start playing with their own peers. You can share the resources on this virtual machine with others to keep them sandboxed. Just destroy the virtual machine when you don’t need the peer anymore or add and remove resources as you like. The launchers download and install the free VirtualBox desktop hypervisor if you don’t already have it installed to run the Peer VM.

    The software implements several layers of security to make sure tenants are isolated from each other and the host peer on which they run. The sandboxes and PGP keys used to manage them are showing in successive layers in the diagram below:

    1. Each container is a sandbox for the software that runs within it. Each container has its own PGP based identity and key pair.
    2. There are two different kinds of security restrictions on resource hosts. Each resource host, a machine, physical or virtual, is a sandbox in itself. Usually the resource host is a virtual machine. Containers in the same cloud environment running on a single resource host have their own sandbox since more than one tenant may reside on the resource host.
    3. Each peer isolates tenants running on its resource hosts. Network communications are encrypted even in the peers LAN.
    4. The peer itself is a sandbox which peer owners manage using a PGP identity.

    Many people ask the question, “When I install a Subutai peer and share resources will others I share with be able to see the things I have on my computer?” Absolutely not. There are so many layers of isolation that there is no real way to have those sharing resources jump out of sandbox after sandbox to get into your computer. Nor are other tenants able to see what’s inside your containers. Obviously the entire basis to Subutai is based on security. Several years were spent just on ensuring tenant isolation with a decentralized identity management system.

Tray Application

This is an OS tray application for Mac OS, Windows, and Linux for desktop integration. It lists and notifies you of situations with your account, your cloud environments and your local and remote peers. The tray is also a wallet application. It shows how much goodwill you have available and notifies you when you spend it, or when you earn more. It works in tandem with the E2E browser security plugins.

E2E Security Plugin and Chrome

There’s a lot of security going on behind the scenes to make everything work safely. Besides usernames and passwords, Subutai uses Pretty Good Privacy (PGP) to do anything important. PGP can be complicated and no one wants to open up terminals to use PGP keys for every action (i.e. manage a peer or cloud environment etc). These browser plugins (extensions) were made to work automatically on your behalf without getting in your way while logging everything that do.

The Launcher wizard automatically installs Chrome and the E2E plugin for it for you. It uses Chrome because it’s available across all platforms. However, the plugin is available for Safari, Chrome, and Firefox. Edge support is coming soon.

P2P Daemon

The Launcher installs a program that runs in the background called the P2P daemon. This daemon allows you to connect to your cloud environments and get on the machines running in them. It works with the tray application and indirectly with the browser plugins to give you a smooth experience with your peers and your cloud environments.
